JR Validated Environment

Controlled R and Python scripts for FDA-regulated medical device teams

Reproducible analysis, audit ready

A framework for running validated R and Python scripts in a controlled, reproducible environment — designed for medical device development teams working under FDA and ISO 13485 requirements.

Get started → Script Guide
What it does

Pinned package versions

Every R and Python package is version-pinned and installed from a controlled local repository — never the internet at runtime.

Consistent across machines

Every team member runs exactly the same packages regardless of their individual machine setup. Shared via Dropbox — no IT infrastructure required.

Integrity checking

Project integrity is SHA256-verified before every script run. Scripts halt immediately if called outside the validated environment.

Audit-ready evidence

Generate a timestamped IQ validation evidence file suitable for an FDA submission or ISO 13485 audit with a single command.

How it works
1

Admin installs once

The administrator specifies package versions in plain text files, builds a local package repository, and stores it in a shared Dropbox folder. This happens once — or whenever packages need updating.

2

Dropbox syncs automatically

The local package repository is distributed to the team through Dropbox. No IT infrastructure, no registries, no internet access required at runtime.

3

Users run scripts

Each team member runs setup_jr_path.zsh once to add the environment to their PATH. On first run of any script the environment builds automatically — all subsequent runs are fast.

Why not Docker?
JR Validated Environment Docker
Learning curve Low — basic Terminal only High — images, registries, Dockerfile
Audit transparency High — plain text requirements files Moderate — binary image requires tooling
Resource usage Minimal — no background processes Heavy — Linux VM always running
Package updates Edit one file, auto-propagated Rebuild and redistribute entire image

Choose the JR Validated Environment if your team is macOS-based, consists of analysts rather than software engineers, and you want validation evidence in plain text files that a Quality Manager can read directly.
