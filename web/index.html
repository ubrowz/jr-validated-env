What it does

Pinned package versions

Every R and Python package is version-pinned and installed from a controlled local repository — never the internet at runtime.

Consistent across machines

Every team member runs exactly the same packages regardless of their individual machine setup. Shared via Dropbox — no IT infrastructure required.

Integrity checking

Project integrity is SHA256-verified before every script run. Scripts halt immediately if called outside the validated environment.

Audit-ready evidence

Generate a timestamped IQ validation evidence file suitable for an FDA submission or ISO 13485 audit with a single command.

How it works
1

Admin installs once

The administrator specifies package versions in plain text files, builds a local package repository, and stores it in a shared Dropbox folder. This happens once — or whenever packages need updating.

2

Admin arranges folder sharing

The team gains access to the local package repository through shared folders. No IT infrastructure, no registries, no internet access required at runtime. See the FAQ for shared folder solutions.

3

Users run scripts

Each team member runs setup_jr_path.zsh once to add the environment to their PATH. On first run of any script the environment builds automatically — all subsequent runs are fast.
