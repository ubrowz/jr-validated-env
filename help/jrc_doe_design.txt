
NAME
	jrc_doe_design — Generate a Design of Experiments (DoE) run matrix

USAGE
	jrc_doe_design "type" "factors_file" "response_name" "output_folder" [centre_points] [replicates]

DESCRIPTION
	Generates a randomised DoE run matrix and writes a self-contained HTML
	file suitable for use on the bench. The HTML contains a Design Summary,
	a Factor Definitions table, and a Randomised Run Order table with a blank
	Response column ready to fill in during the experiment. The file can be
	printed directly from the browser using the Print button.

	Supported design types are 2-level full factorial, 3-level full factorial,
	2-level fractional factorial (minimum resolution), and Plackett-Burman
	screening. The run order is randomised using a time-derived seed that is
	recorded in the output so the sequence is reproducible.

ARGUMENTS
	type           Design type. Must be one of:
	                 full2      — 2-level full factorial
	                 full3      — 3-level full factorial
	                 fractional — 2-level fractional factorial (minimum resolution)
	                 pb         — Plackett-Burman screening
	factors_file   Path to a CSV file defining the factors. See FACTORS FILE.
	response_name  Name of the response variable (e.g. "SealStrength_N").
	               Used as the column header in the run matrix.
	output_folder  Path to the folder where the HTML file is saved.
	               The folder must already exist.
	centre_points  (optional) Integer >= 0. Number of centre-point runs to add.
	               Default: 0. Applies to full2 and fractional only; ignored
	               (with a warning) for full3 and pb.
	replicates     (optional) Integer >= 1. Number of times to replicate the
	               base design. Default: 1. Not applicable to pb designs.
	-h | --help    Show this help message.

DESIGN TYPES
	full2
	  Two-level full factorial. Tests all 2^k combinations of low (-1) and
	  high (+1) factor settings. Provides complete main-effect and interaction
	  information. Run count grows quickly: use fractional for k > 5.
	  Maximum supported: 2^8 = 256 runs (k <= 8 without replicates).

	full3
	  Three-level full factorial. Tests all 3^k combinations of low, centre,
	  and high settings. Allows detection of quadratic (curvature) effects.
	  Maximum supported: 3^5 = 243 runs (k <= 5).

	fractional
	  Two-level fractional factorial generated by FrF2 at minimum resolution.
	  Suitable for screening many factors with fewer runs than a full factorial.
	  Some interactions are aliased with main effects; review the alias
	  structure before interpreting results. Add centre_points to check for
	  curvature without revealing which factor causes it.

	pb
	  Plackett-Burman screening design. Efficient for screening k factors in
	  approximately k+1 runs (rounded up to the nearest multiple of 4).
	  Main effects are partially confounded with two-factor interactions;
	  use only for initial screening. centre_points and replicates are not
	  applicable.

FACTORS FILE
	A plain CSV file with the following columns (case-insensitive):

	  name   — factor name as it will appear in the run matrix
	  low    — actual value at the low (-1) level
	  high   — actual value at the high (+1) level
	  mid    — (optional) actual value at centre (0). If absent, computed as
	           (low + high) / 2.

	Example (seal_factors.csv):
	  name,low,high
	  Temperature_C,140,180
	  Pressure_bar,2.5,4.5
	  DwellTime_s,1.0,3.0

	One row per factor. Blank rows and extra columns are ignored. Factor names
	may contain letters, numbers, and underscores.

CENTRE POINTS
	Centre points are additional runs at the midpoint of every factor (coded
	value 0). They serve two purposes:

	  1. Curvature check — if the response at the centre differs from the
	     average of the factorial responses, a quadratic term is present
	     and a response surface design (e.g. full3) should be considered.
	  2. Pure error estimate — replication at the centre provides an
	     independent estimate of experimental error.

	A typical recommendation is 3 to 5 centre points per experiment.
	Centre points are interspersed randomly throughout the run order so
	they are distributed across any time-related drift.

OUTPUT
	A single self-contained HTML file is saved to output_folder with the name:

	  doe_design_<type>_<response_name>_<YYYYMMDD_HHMMSS>.html

	The file has no external dependencies and can be opened in any browser,
	emailed, or archived. It contains:

	  - Design Summary — type, response, factor count, run counts, seed,
	    and generation timestamp.
	  - Factor Definitions table — factor names with low, (centre,) and high
	    actual values.
	  - Randomised Run Order table — one row per run, showing the actual
	    factor settings (with coded values in grey), a Std Order reference,
	    and a blank Response column to fill in at the bench.
	    Centre-point rows are highlighted in light blue with "CP" in the
	    Std Order column.
	  - Print button — produces a clean, single-page printout suitable for
	    a paper traveller in the laboratory.

EXAMPLES
	1. Three-factor 2-level full factorial with 3 centre points:

	   jrc_doe_design full2 seal_factors.csv SealStrength_N ~/doe_output 3 1

	   Produces: 8 factorial runs + 3 centre points = 11 total runs.

	2. Four-factor fractional factorial (screening, no centre points):

	   jrc_doe_design fractional factors4.csv Torque_Nm ~/doe_output

	   Produces: FrF2 minimum-resolution fractional design, run count
	   determined automatically by FrF2 for 4 factors.

	3. Eight-factor Plackett-Burman screening:

	   jrc_doe_design pb factors8.csv BurstPressure_kPa ~/doe_output

	   Produces: 12-run PB design (ceiling((8+1)/4)*4 = 12).

NOTES
	Run count limits: full2 is capped at 256 runs (k <= 8 for replicates = 1).
	full3 is capped at 243 runs (k <= 5). For larger experiments use
	fractional or pb.

	Seed reproducibility: the randomisation seed is derived from the system
	clock at the time the script runs and is printed to the terminal and
	embedded in the HTML footer. To reproduce the exact run order for
	documentation purposes, record the seed from the HTML file.

	After generating the design, run admin/admin_create_hash to update the
	project integrity file before committing.
