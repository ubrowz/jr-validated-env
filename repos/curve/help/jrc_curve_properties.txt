
What this script does:
       This script extracts engineering-relevant properties from an XY
       measurement series — for example, force vs. displacement, torque
       vs. angle, or pressure vs. volume. The input is a two-column CSV
       and a plain-text .cfg configuration file. Properties are printed
       to the terminal and written to a results text file. An optional
       plot is saved as a PDF.

       The input is treated as a time-ordered measurement series, not a
       mathematical function. The same X value may appear multiple times
       (e.g. loading and unloading arms of a compression curve both pass
       through X = 15 mm). Named phases — defined as row ranges anchored
       by physical X values — resolve this ambiguity.

       Smoothing (Savitzky-Golay or moving average) is applied only to
       derivative-based calculations: instantaneous slope, inflections,
       and yield point. Peak values (max/min), area under curve, and
       hysteresis are always computed on raw data. The results file
       identifies which method was used for each reported value.

Description: Config-file driven XY curve analysis. Extracts peak values,
             AUC, slopes (overall, secant, instantaneous), query values
             (Y at X, X at Y), inflection points, and hysteresis area.
             Saves a results .txt and optional PDF plot.

Usage: jrc_curve_properties path/to/config.cfg

Arguments:
       config.cfg      Path to a .cfg file (INI format) that specifies
                       the data file, column names, phases, and which
                       properties to compute. All data and output paths
                       in the config are relative to the config file's
                       own directory.
       -h | --help     Show this help message.

Config file sections:
       [data]          required
         file          Path to the CSV file (relative to config file).
         x_col         Name of the X column (e.g. displacement_mm).
         y_col         Name of the Y column (e.g. force_N).

       [output]        optional
         label_x       Axis label for X on the plot.
         label_y       Axis label for Y on the plot.
         title         Plot and results title.
         plot          yes | no  — generate the PDF plot.
         plot_file     Output path for the plot (default: <cfg>_plot.pdf).
         results_file  Output path for results (default: <cfg>_results.txt).

       [smoothing]     optional
         method        savgol (default) | moving_avg | none
         span          Fraction of data length used as window (default: 0.15).
                       Larger span = more smoothing. Affects derivatives only.
         apply_to_plot yes | no  — overlay smoothed curve on plot.

       [phase.NAME]    optional, repeatable (e.g. [phase.loading])
         x_start       X value marking the start of this phase. Matched to
                       the closest data point in the full time series.
         x_end         X value marking the end of this phase. Matched to the
                       closest data point at or after x_start.

       [global]        optional
         max_y         yes — global (or phase) maximum Y and its X position.
         max_y_phase   Phase name for max_y (omit for full dataset).
         min_y         yes — global (or phase) minimum Y and its X position.
         min_y_phase   Phase name for min_y (omit for full dataset).
         auc           yes — area under curve (trapezoid rule, raw data).
         auc_phase     Phase name for AUC (omit for full dataset).
         hysteresis    yes — area between two arms (linear interpolation,
                       raw data). Requires both loading and unloading phases.
         hysteresis_loading_phase    Phase name (default: loading).
         hysteresis_unloading_phase  Phase name (default: unloading).

       [slope]         optional
         overall       yes — linear regression slope and R² over the phase.
         overall_phase Phase name (omit for full dataset).
         secant        yes — slope of secant line between secant_x1 and secant_x2.
         secant_phase  Phase name (omit for full dataset).
         secant_x1     X value for first secant point.
         secant_x2     X value for second secant point.
         at_x_1        X value for instantaneous slope (numerical derivative,
                       smoothed data). Add at_x_2, at_x_3, ... for more.
         at_x_1_phase  Phase name for at_x_1 (omit for full dataset).

       [query]         optional
         y_at_x_1      X value — returns interpolated Y at that X.
         y_at_x_1_phase  Phase name (omit for full dataset).
         x_at_y_1      Y value — returns X where curve crosses that Y.
         x_at_y_1_phase  Phase name (omit for full dataset).
         x_at_y_1_mode   first | last | all  (default: first).
                         'all' returns one row per crossing.
         Add _2, _3, ... suffixes to repeat each query type.

       [transitions]   optional
         inflections           yes — X,Y positions where second derivative
                               changes sign (smoothed data).
         inflections_phase     Phase name (omit for full dataset).
         inflections_min_gap   Minimum X spacing between reported inflections.
                               Prevents noise from generating many closely-spaced
                               sign changes. Default: 5% of the phase X range.
                               Set to 0 to report all sign changes.
         yield_slope           Fraction of max slope at which to report the
                               yield-like point (e.g. 0.10 = 10% of max slope).
         yield_phase           Phase name (omit for full dataset).

Config rules:
       - Absent key = skip that feature entirely.
       - All _phase keys tie the analysis to a named phase; omit to use
         the full dataset.
       - Numbered suffixes (_1, _2, ...) allow multiple instances of
         the same query or slope type.
       - Paths in the config are relative to the config file's directory.
       - Decimal separator is always '.' regardless of locale.

Output:
       Terminal    Aligned table of all computed properties with
                   notes identifying raw vs. smoothed source.
       results.txt Tab-aligned text file (default: <cfg>_results.txt).
       plot.pdf    PDF plot showing raw data, optional smoothed curve,
                   phase boundaries, and annotated properties
                   (default: <cfg>_plot.pdf; only if plot = yes).

Example:
       jrc_curve_properties analysis/sample.cfg
       jrc_curve_properties --help
