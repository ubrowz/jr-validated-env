
What this script does:
       Answers the question: "Is this gauge worth using at all?"

       Before running a full Gauge R&R with multiple operators, you want to
       check a more basic question first: is the gauge itself consistent and
       accurate? A Type 1 study does exactly that.

       One person picks up a part whose true value is already known — measured
       by a reference standard or a calibration lab — and measures it 25 to 50
       times, back to back, under identical conditions. No other variables. Just
       the gauge.

       From those repeated measurements you get two numbers:

         Cg  — is the gauge consistent?
               Compares the gauge's natural scatter (6 sigma) to 20% of your
               tolerance. If 6 sigma fits comfortably within that window, the
               gauge is repeatable enough to be useful. Target: Cg >= 1.33.

         Cgk — is the gauge also accurate?
               A gauge can be perfectly consistent but always read 0.1 mm too
               high. Cgk catches that — it applies a penalty for bias on top
               of the repeatability check. A gauge can pass Cg and fail Cgk.
               That tells you the gauge is precise but not accurate.
               Target: Cgk >= 1.33.

       The run chart adds a time dimension: are the readings randomly scattered
       around the reference value, or do they drift upward over the first 10
       measurements? A drift or trend in the run chart is a red flag even if Cg
       looks fine — it can indicate the gauge warms up, the operator's technique
       changes, or the part deforms under repeated clamping.

       The shaded band on the run chart is the acceptance window: reference
       +/- 10% of tolerance. Measurements should stay within this band with no
       visible trend or pattern.

       Rule of thumb: run a Type 1 study before committing to a full Gauge R&R.
       If the gauge fails the Type 1, fix the gauge first — there is no point
       running the bigger study with a gauge that cannot even repeat on itself.

Description: Type 1 Gauge Study (AIAG/VDA method). Computes Cg, Cgk, %Var,
             and %Bias from repeated measurements of a single reference part.
             Performs a t-test for bias significance. Reference value and
             tolerance are required arguments. Saves a run chart and histogram
             PNG to ~/Downloads/.

Usage: jrc_msa_type1 <data.csv> --reference <value> --tolerance <value>

Arguments:
       data.csv              CSV file with a 'value' column containing the
                             repeated measurements. An optional 'id' column
                             is used as the run chart x-axis; if absent,
                             observation order (1, 2, 3 ...) is used.
       --reference <value>   Known true value of the reference part (required).
       --tolerance <value>   Process tolerance USL - LSL (required).
       -h | --help           Show this help message

Input CSV format (minimal):
       value
       10.012
       10.008
       10.015
       ...

Input CSV format (with id):
       id,value
       1,10.012
       2,10.008
       3,10.015
       ...

       Minimum: 10 measurements. Recommended: 25-50.

Output — Terminal:
       Descriptive statistics (mean, SD, bias, 95% CI, bias t-test)
       Cg and Cgk with verdict
       %Var and %Bias
       Pass/fail verdict

Output — PNG (~/Downloads/):
       Left panel:  Run chart — values over time with reference line, mean
                    line, and +/-0.1xT acceptance band (shaded)
       Right panel: Histogram with normal curve overlay and the same
                    reference / acceptance band lines

Acceptance criteria (AIAG/VDA):
       Cg  >= 1.33   Acceptable
       Cg  1.00-1.33 Marginal
       Cg  < 1.00    Unacceptable
       Cgk >= 1.33   Acceptable
       Cgk 1.00-1.33 Marginal
       Cgk < 1.00    Unacceptable

Examples:
       jrc_msa_type1 data.csv --reference 10.000 --tolerance 0.5
       jrc_msa_type1 data.csv --reference 25.4   --tolerance 1.0
